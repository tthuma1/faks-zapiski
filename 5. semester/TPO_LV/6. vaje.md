UML - unified modeling language (use case diagrami):
- pri user storyjih imamo dva tipa - napišemo user story ali pa napišemo elemente
- pri use case je podobno, imamo dva tipa - opiše model ali elemente modela
- pri UML imamo neka pravila za uporabo notacij, potem pa si lahko po svoje razložiš kako boš risal

- s sledenjem standardu skrbimo za doseganje nivoja kakovosti pri risanju diagramov
- imamo več verzij UML jezika
- uporabniška dokumentacija - kaj je narejeno:
	- kako naj jo uporabljamo
- razvojna dokumentacija - kako je narejeno:
	- kako je razvito, da lahko razvijamo/vzdržujemo
- za prikazovanje uporabljenih tehnologij je uporaben component diagram
- statični diagrami:
	- use case
- dinamični diagrami:
	- sequence - pokažeš gibanje podatkovnih tokov
	- component

- use case:
	- akter - uporabnik sistema, ni nujno človek (npr. baza, licenčni stržnik)
	- primer uporabe
	- med akterjem in primerom uporabe delaš povezave:
		- to so asociacije; lahko usmerjena ali ne
	- črtkana črta - generalizacija (dedovanje)
	- include/extend
	- include:
		- npr. vedno kadar pijem, bruham
	- extend:
		- rabimo pogoj, npr. bruham, kadaj pijem nekaj specifičnega
		- tega pogoja ne rabimo posebej napisati v use case diagram - če rabimo več informacij, uporabimo class (razredni) diagram ali kateri drugi diagram - use case diagrami morajo biti enostavni in razumljivi

- včasih je bolje samo z besedo opisati, namesto da delaš diagrame, če ne rabimo high level pregleda
- ontologija - glede na podatkovni tip pokažemo, kaj se dogaja
- https://plantuml.com/
- če imamo kompleksen sistem z veliko podrobnostmi za prikazati, je to bolje narediti tekstovno kot z use-case diagramom
